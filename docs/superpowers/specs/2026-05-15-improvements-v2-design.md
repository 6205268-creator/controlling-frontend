**Дата:** 2026-05-15  
**Статус:** Draft

---

## Цель

Устранить замечания по существующим страницам: навигация с дашборда, редактирование участков и счётчиков, переименование раздела контрагентов с фильтрацией, переименование журнала и добавление отмены операций.

## Архитектурный подход

Все изменения — на фронтенде. API бэкенда не меняется. Используем существующие эндпоинты: `PATCH /plots`, `POST /meters`, `PATCH /meters`, `POST /rpc/cancel_document`. Новых RPC не требуется.

Стек без изменений: React 18 + TypeScript + Tailwind + shadcn/ui + lucide-react.

## 2. Участки — редактирование

**Что:** в каждой строке таблицы кнопка "Редактировать" → диалог редактирования участка.

**Поля диалога:**
| Поле | Тип | Обязательно |
|------|-----|-------------|
| Номер участка | text input | да |
| Площадь (сотки) | number input | да |
| Активен | checkbox | да |

**API:** `PATCH /plots?id=eq.<uuid>` с телом `{ number, area, is_active }`.

**UX:**
- Кнопка "Редактировать" — иконка карандаша (`Pencil` из lucide) в последней колонке строки
- Диалог — modal (shadcn Dialog)
- После сохранения — перезагрузка списка
- Смена владельца — отдельный flow через OwnershipDialog (без изменений)

> **ВАЖНО:** `owner_id` через этот диалог не редактируется.

## 3. Счётчики — добавление и редактирование

### 3.1 Добавление

**Что:** кнопка "Добавить счётчик" на странице → диалог добавления.

**Шаги в диалоге:**
1. Выбрать владельца (поиск по контрагентам, `rpc/search_contractors`). После выбора — запрос `/plot_summary?owner_id=eq.<id>`. Если участков нет → показать ошибку "У этого контрагента нет участков" и не давать продолжить.
2. Система подтягивает участки владельца (`/plot_summary?owner_id=eq.<id>`)
   - 1 участок → `plot_id` подставляется автоматически, поле disabled
   - 2+ участка → выпадающий список участков владельца
3. Тип счётчика — select из enum: Вода / Электричество / Газ
4. Серийный номер — text input

**Все 4 поля обязательны.**

**API:** `POST /meters` с телом `{ organization_id, plot_id, meter_type, serial_number, is_active: true }`.

### 3.2 Редактирование

**Что:** кнопка "Редактировать" в строке счётчика → диалог редактирования.

**Редактируемые поля:**
| Поле | Тип |
|------|-----|
| Тип счётчика | select (вода/электричество/газ) |
| Серийный номер | text input |
| Активен | checkbox |

**API:** `PATCH /meters?id=eq.<uuid>` с телом `{ meter_type, serial_number, is_active }`.

> **ВАЖНО:** `plot_id` и `owner` через редактирование не меняются.

## 5. Журнал операций

### 5.1 Переименование

| Было | Стало |
|------|-------|
| "Журнал документов" (сайдбар, заголовок) | "Журнал операций" |

### 5.2 Отмена операции

**Что:** в каждой строке журнала кнопка "Отменить" — только для проведённых операций (статус `posted`).

**UX:**
- Кнопка "Отменить" (иконка `X` или `Ban`) — в строке
- Перед выполнением — диалог подтверждения: "Отменить операцию? Это действие нельзя отменить."
- После успеха — перезагрузка журнала

**API:** `POST /rpc/cancel_document` с телом `{ p_doc_id: "<uuid>" }`.

> **ВАЖНО:** кнопка "Отменить" не показывается для операций со статусом `cancelled`.
