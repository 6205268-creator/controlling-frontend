# Spec: Множественные владельцы с долями — OwnershipDialog

**Дата:** 2026-05-20  
**Статус:** Approved

---

## Цель

Переработать `OwnershipDialog` для поддержки нескольких владельцев с указанием количества долей у каждого. Исправить существующие баги API. Добавить цветовые темы (CSS-переменные).

---

## Архитектурный подход

Диалог переходит от модели «один владелец» к модели «документ-заголовок + N строк владельцев». Соответствует реальной структуре API: `create_ownership` создаёт заголовок, `add_ownership_owner` / `update_ownership_owner` / `remove_ownership_owner` управляют строками.

Состояние владельцев хранится локально как массив `OwnerEntry[]`. При сохранении — диффинг: новые строки → `add`, изменённые → `update`, удалённые → `remove`.

## Дизайн UI

### Структура диалога (два столбца)

```
┌─────────────────────────────────────────────────────┐
│ [Заголовок — градиент темы]    Право владения    ×  │
├──────────────────────────────────┬──────────────────┤
│ Дата          Участок            │  Доли            │
│                                  │  [Пончик-диагр.] │
│ ── Владельцы ──                  │  • Имя   50%     │
│ ┌──────────────────────┐         │  • Имя   50%     │
│ │ [Пикер] [+] [×]      │         │                  │
│ │ Долей: [1] ████ 50%  │         │  Всего: 2        │
│ └──────────────────────┘         │                  │
│ ┌──────────────────────┐         │                  │
│ │ [Пикер] [+] [×]      │         │                  │

└─────────────────────────────────────────────────────┘
```

### Строка владельца

- **Пикер** (ContractorPicker) — поиск по ФИО  
- **Кнопка [＋]** — открывает мини-попап создания нового контрагента  
  - Нейтральное состояние: серая рамка  
  - Активное (попап открыт): градиент темы, тень  
- **Кнопка [×]** — удалить строку. У единственной строки — скрыта  
- **Поле долей** — `maxLength=1`, только цифра 1–9, акцентная рамка темы  
- **Прогресс-бар** — % от суммы всех долей, цвет темы  
- **% справа** — числом, цвет темы

### Панель долей (правая колонка)

- Круговая диаграмма (CSS conic-gradient, динамическая)
- Легенда: точка цвета + сокращённое имя + %
- Итого: сумма всех долей
- Не выбранный контрагент отображается серым

### Начальное состояние (новый документ)

Одна пустая строка владельца открыта сразу. Кнопка [×] скрыта. «Провести» заблокирована до выбора хотя бы одного контрагента.

### Мини-попап создания контрагента

Открывается как абсолютно-позиционированный слой внутри диалога, прикреплённый к строке владельца (position: absolute, top: bottom строки, left: 0), поверх остального контента диалога (z-index выше). На мобильных — центрирован в диалоге. Закрывается кликом вне попапа или кнопкой «Отмена». Содержит:
- Заголовок с градиентом темы: «Новый контрагент»
- Два компактных тайла (стиль как у счётчиков в настройках, h-14 w-full):
  - 🧑 **Физлицо** — активный: градиент темы (Физлицо-цвет)
  - 🏢 **Юрлицо** — активный: второй градиент темы (Юрлицо-цвет)
  - Неактивный: серый, иконка приглушена
- Поле **ФИО** / **Название организации** (зависит от типа)
- Поле **Телефон**, placeholder: `+375 (__) ___-__-__`
- Кнопки: `[Отмена]` `[Создать]`
- После «Создать»: контрагент сохранён → попап закрыт → пикер строки заполнен

## API — исправления и новые функции

### Исправленные баги в текущем коде

| Функция | Баг | Исправление |
|---------|-----|-------------|
| `createOwnership` | Шлёт `p_contractor_id`, API не принимает | Убрать |
| `updateOwnership` | Шлёт `p_own_id`, API ждёт `p_document_id` | Исправить |
| `getOwnershipByDocumentId` | `limit=1` — только первый владелец | Убрать лимит |
| `postOwnership` | Шлёт `p_doc_id`, API ждёт `p_document_id` | Исправить |
| `unpostOwnership` | Шлёт `p_own_id`, API ждёт `p_document_id` | Исправить |

### Новые функции в api.ts

```typescript
// Добавить владельца к документу
addOwnershipOwner(params: {
  documentId: string
  contractorId: string
  shares?: number
}): Promise<RpcResult>

// Обновить строку владельца (все три поля обязательны в API)
updateOwnershipOwner(params: {
  ownId: string         // own_id строки владельца
  contractorId: string
  shares: number
}): Promise<RpcResult>

// Удалить строку владельца
removeOwnershipOwner(ownId: string): Promise<RpcResult>

// Загрузить ВСЕ строки владельцев документа
getOwnershipOwners(documentId: string): Promise<OwnershipLine[]>
```

### Обновить тип OwnershipLine

`shares: number` уже есть — оставить.

## Логика сохранения

### Новый документ

1. `create_ownership` → получить `doc_id`
2. Для каждого владельца → `add_ownership_owner(doc_id, contractor_id, shares)`
3. Опционально → `post_ownership(doc_id)`

### Редактирование черновика

1. `update_ownership(document_id, doc_date, object_id, notes)`
2. Диффинг массива владельцев:
   - Новые (нет `own_id`) → `add_ownership_owner`
   - Изменённые (есть `own_id`, изменился contractor или shares) → `update_ownership_owner`
   - Удалённые (были в загруженных, нет в текущем списке) → `remove_ownership_owner`
3. Опционально → `post_ownership(document_id)`

## Типы в компоненте

```typescript
interface OwnerEntry {
  localId: string          // uuid для React key, генерируется на клиенте
  ownId: string | null     // null = новый, string = из базы (для update/remove)
  contractor: Contractor | null
  shares: number           // 1–9
}
```

## CSS-переменные тем

Две темы через `[data-theme]` на `<html>`:

**`garden`** (садоводческое товарищество):
- Заголовки: `linear-gradient(135deg, #14532d, #16a34a)`
- Акцент: `#22c55e`, мягкий фон: `#f0fdf4`
- Тайл Физлицо: зелёный градиент
- Тайл Юрлицо: оливковый/лаймовый градиент

**`garage`** (гаражный кооператив):
- Заголовки: `linear-gradient(135deg, #1c1917, #44403c)` (мокрый асфальт)
- Акцент: `#78716c`, мягкий фон: `#f5f5f4`
- Тайл Физлицо: антрацитовый градиент
- Тайл Юрлицо: тёплый серый градиент

По умолчанию применяется `garden`. Тема задаётся через CSS-переменные в `src/index.css` на `:root` (всегда `garden` для текущего проекта). Переключение через `data-theme` на `<html>` заложено для будущих настроек организации — в этом релизе только `garden`.

## Валидация (блокировки)

| Условие | Эффект |
|---------|--------|
| Нет выбранного участка | «Провести» и «Сохранить» — disabled |
| Нет ни одного контрагента | «Провести» — disabled |
| Доли = 0 у любого владельца | Не допускать (minLength=1) |

---

## Режим редактирования (edit mode)

При открытии с `editDocumentId`:
1. Загрузить заголовок документа
2. Загрузить ВСЕ строки (`getOwnershipOwners`) — без `limit=1`
3. Каждую строку преобразовать в `OwnerEntry` с заполненным `ownId`
4. Если статус не `draft` — показать ошибку, не давать редактировать

---

## Out of scope

- Смена темы через UI (только через код/конфиг)
- Максимальное количество владельцев (не ограничено явно, по факту обычно 1–3)
- Дробные доли (API принимает только integer)
